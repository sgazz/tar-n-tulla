# Core Loop

## Moment-to-Moment Loop

1. **Read** the next platform and the current rope state (slack, tension, swing).
2. **Position** Tar and Tulla through tilt and brief air influence.
3. **Time** a jump or landing so one character anchors while the other swings or pulls.
4. **Transfer momentum** through the elastic rope—sling, rebound, or recover.
5. **Stabilize** on a platform or in a controlled hang before the next move.

This loop repeats every 1–3 seconds. The player is always balancing upward progress against rope tension and falling risk.

## Short Session Loop

1. Start a run from the current checkpoint or base height.
2. Climb through a sequence of platforms using the core movement vocabulary.
3. Build score through height, clean landings, and momentum chains.
4. Fail (fall) or reach a session goal (height milestone, level end).
5. Review result and restart immediately.

Target: a complete session loop under 30 seconds from launch to retry.

## Progression Loop

Early progression is **mastery**, not power.

- Unlock or reach new vertical zones and level variants.
- Introduce platform types and hazards gradually.
- Improve personal bests: max height, best combo, fewest falls.
- Optional: cosmetic identity for Tar and Tulla (later milestone).

The player should feel improvement through skill and rhythm, not through stat upgrades.

## Fail / Retry Loop

1. **Fail condition:** Both characters fall below the death threshold (camera floor or kill zone).
2. **Feedback:** Brief, clear failure moment—no long animations.
3. **Restart:** One tap to retry from checkpoint or run start.
4. **Lesson:** Failure should suggest a cause (early jump, bad landing, too much slack, over-rotation).

The retry loop must be frictionless. Players should want "one more run" within seconds of failing.
